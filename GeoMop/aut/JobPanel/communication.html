class communication.communicator.Communicator(init_conf, id=None, action_func_before=None, action_func_after=None, idle_func=None)¶

Bases: object

Class with communication interface, that provide place for action. Communicator contains:

  • intput - communacion interface (communication.communication.InputComm) for communacion with previous communicator
  • input - communacion interface (communication.communication.OutputComm) for communacion with next communicator
  • function action_func_before, that is possible set by constructor, end is call before resending received data from next communicator
  • function action_func_after, that is possible set by constructor, end is call before resending answer received from previous communicator
action_func_after = None¶

Function for processing of action in class above after sending message

Parameter: Is received message object Return: Message that is return or None

action_func_before = None¶

Function for processing of action in class above before sending message

Parameter: Is received message object Return: tuple in Boolean and message. Boolean is True if action is resend to next communicator and if message is not None is send back instead one returned by resend

run()¶

Infinite loop that is interupt by sending stop action by input, that send all messages to next communicator by standart way.

  1. receive message
  2. func self.action_func_before
  3. resend message to next communicator
  4. receve message from next communicator
  5. func self.action_func_after
  6. answer

Redefine self.action_func_before and self.action_func_after for advanced actions. Use send_action instead, if you want work in real time

job_idle_func()¶

Make job specific action. If is not action pending, run anc_idle_func

For connection over ssh make:
  • Find next unconnected over socket job and send add_job message to it over remote. If job_conn akcion is returned, connect, and continue connecion over socket
For other connections make:
  • if stopping is true, try stop next processes
  • Find first unconnected job and try connect it

Call standart communicator idle function. But only if long action is not already processed it.
